# 净高分析功能设计

## 背景

项目需要新增一个 Revit 净高分析插件。实际模型通常包含建筑模型和结构模型，结构模型多以链接模型存在；建筑模型很多时候没有可用的 Room 或 Space 信息。因此，净高分析不能依赖房间边界作为前提。

本设计采用最终版方案：基于建筑外轮廓生成可调网格，将结构、建筑、机电等净高控制构件投影到网格，按每个网格内的最不利净高生成热力图。

## 目标

- 新增独立插件模块 `PlugHub.ClearHeightAnalysis`。
- 识别或选择建筑外轮廓作为分析区域。
- 按用户设定的网格尺寸生成网格，默认 `1000mm × 1000mm`。
- 支持建筑模型、结构链接模型、机电链接模型参与分析。
- 按构件投影覆盖网格，而不是只用单点射线采样。
- 每个网格记录最低净高值和控制构件。
- 在当前视图生成彩色净高热力图。
- 提供低净高结果表，支持追溯控制构件。
- 支持清理上一次分析结果。

## 非目标

- 第一版不做力学有限元分析。
- 第一版不依赖 Room 或 Space；房间统计只作为后续增强。
- 第一版不做全项目所有楼层批量分析。
- 第一版不保证对所有异形族进行精确实体裁剪。
- 第一版不生成正式审图报告或 Excel 模板报告。

## 用户流程

1. 用户点击 `净高分析`。
2. 弹出设置窗口。
3. 用户选择楼层。
4. 用户选择建筑外轮廓来源：
   - 自动识别建筑楼板外轮廓；
   - 选择楼板；
   - 选择闭合线或填充区域；
   - 使用当前视图裁剪框或手动框选范围作为兜底。
5. 用户选择参与分析的模型：
   - 当前模型；
   - 结构链接；
   - 机电链接；
   - 其他链接。
6. 用户设置分析参数：
   - 网格尺寸；
   - 完成面偏移；
   - 净高阈值；
   - 搜索高度；
   - 是否包含吊顶；
   - 是否包含机电；
   - 最小管径过滤；
   - 显示全部热力图或只显示不合格网格。
7. 用户点击分析。
8. 插件生成热力图和结果表。
9. 用户可清除结果或导出 CSV。

## 核心算法

### 1. 外轮廓生成

`BuildingOutlineProvider` 负责生成分析区域边界。

优先级：

1. 用户选择的闭合边界。
2. 用户选择的楼板外边界。
3. 自动识别指定楼层建筑楼板外轮廓。
4. 自动识别指定楼层结构楼板外轮廓。
5. 当前视图裁剪框或手动框选范围。

输出为一个或多个平面多边形。第一版要求至少支持一个外轮廓，可保留洞口和多轮廓接口，后续增强。

### 2. 网格生成

`GridBuilder` 在分析边界内生成规则网格。

- 默认网格尺寸：`1000mm`。
- 用户可输入自定义尺寸。
- 建筑外轮廓外不生成网格。
- 网格对象包含：
  - 网格编号；
  - XY 范围；
  - 中心点；
  - 所属楼层；
  - 当前最低净高；
  - 控制构件列表。

### 3. 障碍构件收集

`ObstacleCollector` 收集参与分析的构件。

默认类别：

- 结构楼板：`OST_Floors`
- 结构梁：`OST_StructuralFraming`
- 结构柱或柱帽：`OST_StructuralColumns`
- 吊顶：`OST_Ceilings`，可选
- 风管：`OST_DuctCurves`
- 风管附件：`OST_DuctFitting`、`OST_DuctAccessory`
- 桥架：`OST_CableTray`、`OST_CableTrayFitting`
- 管道：`OST_PipeCurves`、`OST_PipeFitting`、`OST_PipeAccessory`

构件来源包括当前文档和链接文档。链接构件需要记录链接实例和坐标变换。

### 4. 构件投影到网格

`ObstacleProjectionMapper` 将每个障碍构件映射到受影响网格。

策略：

1. 获取构件 BoundingBox。
2. 将链接构件 BoundingBox 转换到当前文档坐标。
3. 用 BoundingBox 的 XY 投影快速找到覆盖的网格范围。
4. 按类别做简化精筛：
   - 楼板和吊顶按水平投影面处理；
   - 梁按 BoundingBox 或曲线扫掠范围处理；
   - 风管、桥架、管道按 BoundingBox 投影处理；
   - 圆管后续可用直径和中心线优化。
5. 对每个受影响网格更新最低底标高。

不采用“每格中心点射线”作为主算法，避免漏掉穿过网格边缘的梁、风管、桥架或管道。

### 5. 净高计算

`ClearHeightCalculator` 对每个网格计算最不利净高。

公式：

```text
净高 = 网格内最低控制构件底标高 - 楼层标高 - 完成面偏移
```

每个网格取最小净高，不取平均值。原因是净高分析属于风险检查，局部低点不能被平均值掩盖。

如果网格没有命中任何顶部控制构件，状态为 `未判断`，默认显示为灰色或透明。

## 热力图显示

`HeatmapRenderer` 在当前视图生成彩色网格。

颜色建议：

- 红色：严重不足，`净高 < 阈值 - 300mm`
- 橙色：不足，`阈值 - 300mm <= 净高 < 阈值`
- 黄色：临界，`阈值 <= 净高 < 阈值 + 300mm`
- 绿色：达标，`净高 >= 阈值 + 300mm`
- 灰色：无法判断
- 透明：建筑外轮廓外

结果图元建议使用可批量管理的面片图元。第一版可根据 Revit 2020 API 可行性在 `FilledRegion` 和 `DirectShape` 中择优：

- `FilledRegion`：平面视图直观，适合二维热力图。
- `DirectShape`：更适合三维表达和统一管理，但实现更复杂。

结果图元需要写入分析批次标识，便于清理。

## 结果表

`AnalysisResultWindow` 展示低净高网格和无法判断网格。

字段：

- 楼层
- 网格编号
- 网格中心坐标
- 净高值
- 阈值
- 差值
- 控制构件名称
- 控制构件类别
- 来源模型
- ElementId
- 是否来自链接模型

后续可支持点击结果定位网格和高亮控制构件。

## 结果清理

`ResultCleanupService` 负责清理旧热力图。

- 分析前可提示是否清除旧结果。
- 提供独立 `清除净高分析结果` 操作。
- 删除时只删除带有本插件批次标识的图元。

## 组件设计

### `ClearHeightAnalysisCommand`

Revit 命令入口。负责获取当前文档、打开设置窗口、启动分析事务。

### `ClearHeightAnalysisWindow`

WPF 设置窗口。负责收集楼层、范围、模型、类别和阈值参数。

### `AnalysisSettings`

分析参数对象。包含楼层、网格尺寸、净高阈值、完成面偏移、搜索高度、类别开关和显示模式。

### `BuildingOutlineProvider`

生成分析区域外轮廓。支持自动识别和用户选择兜底。

### `GridBuilder`

根据外轮廓和网格尺寸生成网格。

### `ObstacleCollector`

从当前文档和链接文档收集候选障碍构件。

### `ObstacleProjectionMapper`

将构件投影覆盖到网格，并记录构件底标高。

### `ClearHeightCalculator`

计算每格最小净高、风险等级和控制构件。

### `HeatmapRenderer`

生成彩色热力图图元。

### `AnalysisResultWindow`

展示分析结果表。

### `ResultCleanupService`

清理旧结果图元。

## 错误处理

- 未找到楼层：提示用户选择有效楼层。
- 无法识别外轮廓：引导用户选择楼板、闭合线或手动框选。
- 链接模型未加载：在模型列表中显示不可用状态。
- 没有选择任何参与模型：阻止分析并提示。
- 没有收集到障碍构件：提示用户检查类别和链接模型。
- 网格数量过多：提示增大网格尺寸或缩小范围。
- 结果图元生成失败：保留结果表，并提示视图或图元创建限制。

## 性能策略

- 不做 `网格数 × 构件数` 的全量暴力循环。
- 建立网格空间索引。
- 每个构件只更新其 BoundingBox 覆盖到的网格。
- 对大面积楼层提示用户使用更大的网格尺寸。
- 默认只渲染不合格和临界网格，完整热力图可选。
- 保留进度条和取消接口，第一版可先做同步分析，后续优化为分阶段进度。

## 分阶段实现建议

### 阶段 1：核心热力图

- 新建模块和清单。
- 实现设置窗口。
- 实现手动范围或楼板范围。
- 实现网格生成。
- 实现当前文档和结构链接楼板、梁分析。
- 生成热力图和结果表。

### 阶段 2：机电覆盖

- 增加风管、桥架、管道类别。
- 增加最小管径过滤。
- 优化线性构件投影覆盖逻辑。

### 阶段 3：外轮廓自动识别增强

- 自动识别建筑楼板外轮廓。
- 支持闭合线、填充区域、多轮廓和洞口。

### 阶段 4：成果增强

- 结果定位。
- CSV 导出。
- 完整热力图和只显示问题区域切换。
- 多楼层批量分析。

## 已确认决策

- 采用最终版热力图方向。
- 不依赖房间信息。
- 主算法使用构件投影覆盖网格，不使用单点采样作为主算法。
- 每个网格取最小净高。
- 网格尺寸由用户控制，默认 `1000mm`。
- 结构和机电构件即使只穿过网格边缘，也应影响该网格结果。
